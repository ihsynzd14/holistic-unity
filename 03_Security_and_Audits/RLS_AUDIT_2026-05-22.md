## Metodologia

L'audit combina **3 approcci complementari**:

1. **Static analysis dei migration file** (source-of-truth: il team usa migration-driven schema, ogni cambio prod passa da SQL versioned in git).
2. **Black-box test con anon role** via `curl` contro PostgREST production endpoint (live verification).
3. **Manual SQL checklist** (3 query documentate sotto) da rieseguire periodicamente via Supabase Dashboard SQL Editor — gap noto: PAT locale + Docker assenti per `supabase db dump --linked`, quindi i 3 controlli vanno fatti manualmente.

Nota su perché la live SQL via CLI non è stata possibile in questo audit: `supabase db dump --linked` richiede Docker Desktop running (non disponibile su questa workstation); `psql` diretto richiede DB password che non è in env locale. Il Management API endpoint `/v1/projects/{ref}/database/query` richiede un PAT che non è in nessuno dei due path standard (`~/.supabase/`, `~/AppData/Roaming/supabase/`).

## Query da rieseguire periodicamente via Dashboard SQL Editor

### Q1 — Coverage check (dalla spec del task)

```sql
SELECT n.nspname, c.relname, c.relrowsecurity, c.relkind
FROM pg_class c
JOIN pg_namespace n ON c.relnamespace = n.oid
WHERE n.nspname = 'public'
  AND c.relkind IN ('r','v','m')
ORDER BY c.relkind, c.relname;
```
**Pass**: ogni riga con `relkind='r'` ha `relrowsecurity = true`. Conteggio atteso: 24 row (23 tabelle + 1 — `tos_acceptances` count incluso). View esenti.

### Q2 — Enumerazione policy

```sql
SELECT schemaname, tablename, policyname, permissive, roles, cmd,
       qual AS using_expression, with_check
FROM pg_policies
WHERE schemaname = 'public'
ORDER BY tablename, policyname;
```
**Pass**: zero righe con `qual = 'true'` su tabelle PII (`users`, `therapist_profiles`, `bookings`, `transactions`, `payment_methods`, `messages`, `mfa_*`, `fattureincloud_*`, `therapist_invoices`). Conteggio atteso: ~80-90 policy.

### Q3 — Column-level grant su `anon`

```sql
SELECT table_name, column_name, privilege_type
FROM information_schema.column_privileges
WHERE table_schema = 'public'
  AND grantee = 'anon'
  AND privilege_type = 'SELECT'
ORDER BY table_name, column_name;
```
**Pass**: per `therapist_profiles`, NON deve apparire `codice_fiscale`, `p_iva`, `stripe_connected_account_id`, `pec_email`, `vat_number`, `billing_address`, `billing_email`, `tax_id_foreign`, `regime_forfettario`. Per le altre tabelle PII: zero righe per `anon` SELECT (anon non deve avere GRANT a livello tabella; access controlled by RLS on authenticated).
